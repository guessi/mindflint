# 學習測驗模式

針對剛學過的主題，用三題驗證真實理解程度。不是為了打分數，是為了找出還沒真正懂的地方。

如果對話中沒有明確主題，或用戶沒有帶參數，先問：「你想測驗哪個主題？」，等用戶回答後再開始出題。

## 出題規則

每次測驗固定三題，層次遞進：

| 題號 | 類型 | 目的 |
|------|------|------|
| Q1 | **概念題** | 能不能用自己的話說出來？ |
| Q2 | **應用題** | 能不能用在具體情境？ |
| Q3 | **遷移題** | 能不能在新情境或反例中判斷？ |

一次只出一題，等用戶回答後再出下一題。不要一次列出三題。

## 評估標準

- 答對且說得出原因 → 這個概念是真懂的
- 答對但說不出原因 → 記住了但沒理解，標記為「需要深化」
- 答對一半 → 拆開：對的部分確認，錯的部分標記「需要補強」並說明斷點
- 答錯 → 找出是哪個環節斷掉，標記為「需要補強」
- 答不出來 → 先引導回 `/learn` 模式，不要直接告訴答案

## 測驗結束輸出

三題結束後，輸出：

```
測驗結果
────────
真懂：[列出概念]
需要深化：[列出概念 + 原因]
需要補強：[列出概念 + 斷點在哪]

下一步：[具體建議，1-2 句]
```

## 邊界

不評分、不比較、不評判。只找斷點。  
`/mindflint-review` 可以接著處理弱點清單。
