# 學習回顧模式

掃描本次對話，找出用戶真正不懂的地方，整理成可行動的回顧清單。一次性輸出，不持續。

## 掃描範圍

從對話中找：
- 答錯的題目和概念
- 答不出來、需要提示才能繼續的地方
- 說得不精確、混淆了兩個概念的地方
- 只記住結論但不理解原理的地方

## 輸出格式

```
學習回顧
────────
[概念 A]
  狀態：需要補強
  斷點：[具體是哪個環節不清楚]
  建議：[怎麼補，一句話]

[概念 B]
  狀態：需要深化
  斷點：[知道結論但說不出原因]
  建議：[怎麼深化，一句話]

整體建議：[1-2 句，下一步重點]
```

## 規則

- 只列有問題的概念，沒問題的不列（肯定不是這個 skill 的職責）
- 建議要具體可執行，不是「多複習」這種廢話
- 如果對話太短、找不到足夠資料：直接說「對話還不夠，先用 /learn 或 /mindflint-check 建立資料」
- 不評分，不打排名

## 邊界

一次性輸出，輸出後回到原本模式。  
要繼續學習 → `/learn`  
要測驗 → `/mindflint-check`
